ChromeWave — Extension Architecture

How the pieces fit together

Architecture Overview

Key Points

  • Background Service Worker — handles tabCapture (requires background context). Streams raw audio to the side panel via a message port.
  • Side Panel (React) — all UI lives here: record controls, waveform editor, effects, file management, and recording library.
  • Storage — metadata in chrome.storage.local, audio blobs in IndexedDB (no size limit), final export via chrome.downloads.
  • Settings — WXT options page for configuring default folder name, preferred format, and recording quality.
