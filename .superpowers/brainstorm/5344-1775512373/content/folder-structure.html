Key Design Decisions

  • entrypoints/ — WXT convention. Each subfolder becomes a Chrome extension entry. WXT auto-generates the manifest.
  • components/ — Pure React UI, no business logic. Each maps to a visible section of the side panel.
  • lib/ — Framework-agnostic core logic. Testable without React. Effects are easy to add here — just create a new function in audio-engine.ts.
  • hooks/ — Bridges between lib/ and components/. State management lives here.
  • types/ — Shared TypeScript interfaces used across lib/, hooks/, and components/.
